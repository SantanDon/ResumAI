New Features Implementation Plan
Goal
Implement a comprehensive suite of career tools including a central dashboard, an intelligent CV chat assistant, and a mass mailing system.

User Review Required
Mass Mail Limits: Confirmed 50 emails max per batch and 4-day cooldown per recipient.
Detailed Feature Specifications
1. Main Dashboard (LandingPage.tsx)
Layout: A premium, high-impact landing screen.
Navigation: Three primary action buttons:
Create CV: (Placeholder for future builder).
CV Chat: Enters the AI analysis and enhancement workflow.
Mass Mail: Enters the bulk email outreach tool.
2. CV Chat Workflow (AIChatInterface.tsx)
Step 1: Analysis & Summary
Upon entering (or after upload), AI analyzes the CV.
Display: A card showing "AI Summary of User Overview" (e.g., "Don Santos is a senior software engineer with...").
Step 2: Chat Interface
A conversational interface aware of the CV context.
Quick Actions:
"Enhance My CV": Triggers a review for general improvements (skills, phrasing).
"Tailor for Job": Opens a prompt for the user to paste a job description.
Step 3: Tailoring Result
Input: User provides Job Description.
Process: AI maps CV skills/experience to JD requirements.
Output:
"I have enhanced the following..." message.
Download Link: A PDF of the tailored CV.
Change Log: A list of specific additions/removals and reasons.
3. Mass Mail Workflow (MassMailInterface.tsx)
Step 1: Recipient Management
Input field for multiple email addresses (or CSV paste).
Validation: Check against email_logs DB for the 4-day cooldown.
Limit: Max 50 recipients per batch.
Step 2: Context & Generation
Input: "Who are these recipients?" (e.g., "Hiring Managers at Tech Corps").
Input: "What is this email about?" (e.g., "Application for Senior Dev role").
AI Process: Generates a professional, context-aware email template.
Step 3: Review & Send
User reviews and edits the generated draft.
Action: "Send to [N] Recipients".
Backend: Sends emails individually, logs timestamp in email_logs.
Technical Implementation
Backend (server/src)
Database (db.ts):
Create email_logs table: id, user_id, recipient_email, sent_at.
API Endpoints:
POST /api/cv/summary: Returns the high-level summary.
POST /api/cv/tailor: Accepts JD, returns tailored PDF path and changelog.
POST /api/mail/check-cooldown: Validates a list of emails.
POST /api/mail/generate: Generates email body.
POST /api/mail/send: Sends batch and updates logs.
Frontend (frontend/src)
Routing: Refactor App.tsx to manage views (landing, chat, mail).
Components:
LandingPage: Hero section with 3 main buttons.
AIChatInterface: Split view (Chat + Context/Actions).
MassMailInterface: Stepper wizard (Recipients -> Context -> Review).
Verification Plan
CV Chat: Upload CV -> Verify Summary appears -> Request Tailoring -> Verify PDF generation.
Mass Mail: Add email -> Generate -> Send -> Try adding same email -> Verify blocked (Cooldown).
