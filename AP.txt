AfriPing Bufferbloat Mitigation SpecificationIntroductionThis feature addresses the bufferbloat issue identified in the ExitLag Network Analysis Summary, where latency spikes from 16ms (no load) to 134ms (upload load) with a C-grade score. Bufferbloat occurs when routers buffer too much data, causing high latency under load (e.g., downloads/uploads competing with gaming traffic). For South African gamers on ISPs like Be Broadband (102.67.159.x), this manifests as jitter spikes (3.59ms UDP) and packet loss at ISP hops (85.71% at 100.67.1.11).The solution emulates SQM (Smart Queue Management) on Windows via netsh QoS policies, bandwidth capping, and TCP tweaks. This prioritizes gaming traffic, caps line rate to 90% (reducing spikes), and applies ECN for congestion avoidance. Expected gains: Bufferbloat A/B grade, jitter <5ms, 20-50ms stability under load.Problem AnalysisCurrent StateNo Load Latency: 16ms (good baseline).
Under Load: 67ms download (+51ms), 134ms upload (+118ms)—C grade.
Protocol Jitter: UDP 3.59ms, TCP 5.68ms—acceptable, but spikes to 121ms at ISP hop.
Root Cause: Router/ISP buffers fill during load, delaying game packets (bufferbloat). Windows lacks native SQM (router-level), so app-level emulation needed.

Solution ApproachQoS Prioritization: DSCP 46 (EF class) for game ports (37000 UDP for Apex).
Bandwidth Capping: Limit upload/download to 90% ISP speed (netsh limits).
TCP Window Scaling: Enable ECN, autotuning for high-latency (netsh int tcp set global).
Monitoring: Track latency under load; alert if >30ms spike.
Fallback: Direct tweaks if no router support.

GlossaryBufferbloat: Latency increase from excessive router buffering under load.
SQM: Smart Queue Management—prioritizes low-latency traffic (emulated via netsh).
DSCP: Differentiated Services Code Point—marks packets for prioritization (46 = Expedited Forwarding for gaming).
ECN: Explicit Congestion Notification—reduces retransmits in high-latency paths.
FQ-CoDEL: Fair Queueing with Controlled Delay—algorithm for AQM (emulated in netsh policies).

Requirement 2User Story: As a user, I want TCP optimizations for high-latency SA connections, so that window scaling reduces retransmits and spikes.Acceptance CriteriaWHEN optimizations run THEN the system SHALL enable ECN (netsh int tcp set global ecncapability=enabled).
WHEN optimizations run THEN the system SHALL set autotuning to normal (netsh int tcp set global autotuninglevel=normal).
WHEN optimizations run THEN the system SHALL increase receive window to 65535 (netsh int tcp set global rss=enabled).
WHEN rollback is requested THEN the system SHALL reset to defaults (netsh int tcp set global ecncapability=disabled).

WHEN test shows <30ms spike THEN the system SHALL display "Bufferbloat Fixed (A Grade)".
WHEN rollback requested THEN the system SHALL reset all netsh changes.

#[tauri::command]
pub fn bufferbloat_fix(isp_speed_up: u32, isp_speed_down: u32) -> Result<OptimizationResult, String> {
    let result = BufferbloatFixer::apply_all(isp_speed_up, isp_speed_down)?;
    let metrics = LoadTester::run_test()?;
    Ok(OptimizationResult {
        qos_applied: true,
        tcp_tweaks_applied: true,
        bandwidth_capped: true,
        mtu_set: true,
        rollback_token: generate_rollback_token(),
        pre_test_latency: metrics.no_load_latency,
        post_test_latency: metrics.upload_spike,
        grade: calculate_grade(&metrics),
    })
}

#[tauri::command]
pub fn rollback_bufferbloat(token: String) -> Result<String, String> {
    BufferbloatFixer::rollback(&token)?;
    Ok("Bufferbloat fixes reverted — original settings restored".to_string())
}

pub fn test_bufferbloat() -> Result<LoadTestMetrics, String> {
    LoadTester::run_test()
}
