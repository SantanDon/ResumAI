# Requirements Document

## Introduction

This feature enhances the email connection flow in the Mass Mail Interface by providing an interactive popup/modal that guides users through the process of obtaining an app-specific password from their email provider. The current implementation shows static instructions, but users often struggle with the multi-step process of enabling 2FA and generating app passwords. This feature aims to reduce friction and improve success rates for email connection setup.

## Glossary

- **App Password**: A 16-character password generated by email providers (Gmail, Outlook, Yahoo) that allows third-party applications to access email services without using the user's primary account password
- **2FA (Two-Factor Authentication)**: A security method requiring two forms of verification before granting access to an account
- **SMTP**: Simple Mail Transfer Protocol - the standard protocol for sending emails
- **Mass Mail Interface**: The component in the application that allows users to send personalized emails to multiple recipients
- **Security Helper Modal**: The popup dialog that provides step-by-step guidance for obtaining app passwords

## Requirements

### Requirement 1

**User Story:** As a user connecting my Gmail account, I want an interactive guide that walks me through getting an app password, so that I can complete the setup without leaving the application confused.

#### Acceptance Criteria

1. WHEN a user clicks a "Help me get App Password" button in the Gmail auth section THEN the Security_Helper_Modal SHALL display with Gmail-specific instructions
2. WHEN the Security_Helper_Modal displays for Gmail THEN the Security_Helper_Modal SHALL show a numbered step-by-step wizard with progress indicators
3. WHEN a user is on a step requiring external navigation THEN the Security_Helper_Modal SHALL provide a button that opens the relevant Google page in a new browser tab
4. WHEN a user completes a step THEN the Security_Helper_Modal SHALL allow the user to mark the step complete and advance to the next step
5. WHEN a user reaches the final step THEN the Security_Helper_Modal SHALL provide a field to paste the generated app password with a "Use This Password" button that auto-fills the main form

### Requirement 2

**User Story:** As a user connecting my Outlook account, I want provider-specific guidance for getting an app password, so that I receive accurate instructions for Microsoft's security settings.

#### Acceptance Criteria

1. WHEN a user clicks a "Help me get App Password" button in the Outlook auth section THEN the Security_Helper_Modal SHALL display with Outlook-specific instructions
2. WHEN the Security_Helper_Modal displays for Outlook THEN the Security_Helper_Modal SHALL include links to Microsoft account security pages
3. WHEN a user is viewing Outlook instructions THEN the Security_Helper_Modal SHALL explain that regular passwords may work but app passwords are recommended

### Requirement 3

**User Story:** As a user connecting my Yahoo account, I want provider-specific guidance for getting an app password, so that I receive accurate instructions for Yahoo's security settings.

#### Acceptance Criteria

1. WHEN a user clicks a "Help me get App Password" button in the Yahoo auth section THEN the Security_Helper_Modal SHALL display with Yahoo-specific instructions
2. WHEN the Security_Helper_Modal displays for Yahoo THEN the Security_Helper_Modal SHALL include links to Yahoo account security pages

### Requirement 4

**User Story:** As a user, I want the helper modal to be dismissible and non-blocking, so that I can close it if I already know how to get my app password.

#### Acceptance Criteria

1. WHEN the Security_Helper_Modal is open THEN the Security_Helper_Modal SHALL display a close button in the top-right corner
2. WHEN a user clicks outside the Security_Helper_Modal THEN the Security_Helper_Modal SHALL close
3. WHEN a user presses the Escape key THEN the Security_Helper_Modal SHALL close
4. WHEN the Security_Helper_Modal closes THEN the Mass_Mail_Interface SHALL preserve any previously entered credentials

### Requirement 5

**User Story:** As a user, I want visual feedback showing my progress through the app password setup steps, so that I know how much of the process remains.

#### Acceptance Criteria

1. WHEN the Security_Helper_Modal displays THEN the Security_Helper_Modal SHALL show a progress indicator displaying current step and total steps
2. WHEN a user completes a step THEN the Security_Helper_Modal SHALL update the progress indicator to reflect completion
3. WHEN a user navigates between steps THEN the Security_Helper_Modal SHALL highlight the current step visually
